1 MatrixBerry-Core
2 ------------
3 library was originally developed as a set of auxilary classes and functions for [Ellipsodal Toolbox for Matlab](https://github.com/SystemAnalysisDpt-CMC-MSU/ellipsoids) project
4 back in 2014. Then in 2016 it was decided to separate the library from "Ellipsoidal Toolbox" so that it can be developed as an independent open-source project.
5 
6 Right from the begining we have been following Test Driven Development (TTD) approach by implementing and extending a test coverage
7 for most of the library functions. Our ideology is to cover every new feature with meanigful tests and every bug found - with negative tests.
8 
9 Features
10 ------------
11 * **Configuration management** - Version-tracked hostname-specific configurations stored as plain xml for easier editing and
12 scm system integration
13 * **Containers**
14  - On-disk hash maps in `mxberry.core.cont.ondisk` package.
15  - `mxberry.core.cont.MapExtended` is an extension of built-in `containers.Map` class that supports
16  - conversion to structures
17  - union of hash maps
18  - deep copies
19  - comparison
20 
21 * **Input parameter parsing functions** `mxberry.core.parseparext` and `mxberry.core.parseparams` and input parameters checking functions in `mxberry.core.check` package
22 * **OOP helper classes**
23 
24  - **for handle classes** - `mxberry.core.obj.HandleObjectCloner`
25 class designed as a parent class
26 for handle objects that require such features as
27  - automatic deep copy
28  - comparison, sorting
29  - support for `ismember` and `unique` operations
30 Check out tests in `mxberry.core.obj.test` package for more details
31 
32  - **for emulation of static properties** - `mxberry.core.obj.StaticPropStorage` class emulates static properties via persistent variables.
33 Check out tests in `mxberry.core.obj.test` package for more details
34 
35 * **ismember and unique functions for arbitrary types** - `mxberry.core` package contains a lot of functions with `ismember*` and `unique*`
36 name patterns for different types of unique-like and ismember-like operations.
37 mxberry.core.ismember and mxberry.core.unique use these operations to implement `ismember`
38 and `unique` for arbitrary types.
39 Check out tests in `mxberry.core.test.IsMemberTC` test case for more details
40 
41 * **Easier Matlab exceptions handling** - `mxberry.core.MExceptionUtils` class provides methods for converting Matlab exceptions
42 into plain text and hyper text strings.
43 * **Useful generic-purpose functions in** `mxberry.core` package, such as
44  - `absrelcompare` for comparing arrays using both relative and absolute tolerances
45  - `getcallername` and `getcallernameext` for getting names of callers
46  - `throwerror` and `throwwarn` for throwing warnings and exceptions with automatically generated identifiers
47 
48 * **Helper functions for different built-in types**
49  - `mxberry.core.string` - working with strings
50  - `mxberry.core.struct` - structures
51  - `mxberry.core.cell` - cells
52 
53 * **Utility functions for working with files and directories** are located in `mxberry.io` package. All these functions are implemented
54 in Java so they are free of filename and path length limit problems on Windows platforms.
55 
56 * **Java static path management** - `mxberry.java.AJavaStaticPathMgr` class is designed as extendable abstract class
57 for much easier management of java static path in Matlab, including an automatic
58 deployment of 'jar' files. See `mxberry.selfmnt.JavaStaticPathMgr` class for an
59 example of AJavaStaticPathMgr abstract methods implementation for a deployment of MatrixBerry Core library.
60 
61 * **System information** - `mxberry.system` package contains functions for getting
62  - pid of current Matlab process
63  - computer name
64  - current user name
65  - names and parameters of all network interfaces
66 
67 * **XML serialization and de-serialization of Matlab structures** in `mxberry.xml.xmlformat` package.
68

80 ### Getting Set Up
81 
82 1. Fork this repository on Github.
83 2. Clone *your forked repository* (not our original one) to your hard drive with
84 
85 ```shell
86 git clone https://github.com/YOURUSERNAME/mxberry-core.git
87 ```
88 
89 3. `cd mxberry-core`
90 4. Start Matlab and make sure to run `s_install` script from `install` subfolder. You can do this either manually from Matlab command line or via a shell script for a specific Matlab version in `install` subfolder.
91 
92 ```matlab
93 # either run this from within Matlab
94 cd install;
95 s_install;
96 ```
97 
98 ```shell
99 # OR run this from shell
100 cd install
101 ./start_matlab2016b_glnxa64.sh #for windows platform use a bat script
102 ```
103 Please keep in mind that if you do not use the start script from `install` subfolder to start Matlab you need to make sure that
104 your "Start in" directory is always `mxberry-core/install`. That is because the very first run of `s_install` script creates `javaclasspath.txt` file with absolute paths to some `jar` files that are a part of MatrixBerry-Core library. As part of this very first run the jar files are added to *dynamic Java path of Matlab JVM*. All subsequent Matlab runs with "Start in" directory set to `mxberry-core/install` load the created `javaclasspath.txt` file thus adding the jar files to *static Java path of Matlab JVM*.
